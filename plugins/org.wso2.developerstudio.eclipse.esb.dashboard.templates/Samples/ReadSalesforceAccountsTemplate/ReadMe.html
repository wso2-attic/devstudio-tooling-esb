Fetch Salesforce Leads Template

About

This sample demonstrates how we can integrate WSO2 EI with the SalesForce system.

Records related to sales leads are fetched from the Salesforce system by a service in WSO2 EI (when it receive a request from a client).

This template creates an ESB Config Project, a Connector Exporter Project and a Composite application project.

The ESB config project consists of a Proxy service called ‘SalesforceAccountService’, which is used to connect with the Salesforce system to retrieve the data relates to sales accounts. The Connector exporter project consists of Salesforce connector, which enables these operations in the mediation flow.

When the proxy service gets invoked, it connects to the Salesforce via the connector and responds with the data related to sales accounts.

Executing the sample

  1. Connect the ESB server to WSO2 EI Tooling

    Attach an EI server runtime into the Developer Studio following the instructions given below (if you haven’t already done).

    Open the Developer Studio Dashboard and click Server under Add Server.

    In the ‘Define a New Server’ dialog box, expand the WSO2 folder, and select the respective product and version.

    Click Next, and then click Browse (next to CARBON_HOME) and select the WSO2 EI distribution directory.

    Click Next and review the default ports for the ESB profile of WSO2 EI. Typically, you can leave these unchanged, but if you are already running another server on these ports, specify unused ports here.

    Click Finish. The ESB server of WSO2 EI is now connected to WSO2 EI Tooling.

  2. Deploy the ESB artifacts in the server

    In the Servers tab, right-click the server and click Add and Remove.

    Select the relevant project from the list, and click Add.

    Click Finish. The template sample is now deployed in the server.

  3. Start the server

    Click the Play button on the toolbar.

    If you are prompted to save changes to any of the artifact files, click Yes.

    As the server starts, the Console tab will appear. You should see messages indicating that the C-App was successfully deployed.

  4. Invoke the service with the following request. Use an HTTP client like cURL.

    curl http://localhost:8280/services/SalesforceAccountService -H "Content-Type: text/xml"
